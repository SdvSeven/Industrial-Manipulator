# UX "Индустриальный манипулятор" — Спецификация

---

## 1. Общая концепция

Программа представляет собой **панель оператора промышленного манипулятора** с четырьмя основными разделами:

| Раздел | Назначение |
|---|---|
| Главная    | Подключение устройств и быстрый старт |
| Управление | Реальное управление манипулятором     |
| Настройки  | Конфигурация системы                  |
| Журнал     | Аналитика и отладка                   |

---

## 2. Страницы и функционал

### 2.1 Главная (Home) — «Центр управления подключением»

**Назначение:** проверка системы перед запуском, подключение устройств, быстрый старт.

---

#### 2.1.1 DeviceBar

Горизонтальная панель состояния всех подключённых устройств. Если устройств несколько — выбор из выпадающего списка.

**Формат отображения:**

```
🟢 Arduino (COM3)   🟢 Camera (0)   🟡 MediaPipe   🔴 Connection
```

**Состояния индикаторов:**

| Индикатор | Значение |
|---|---|
| 🟢 | OK — устройство работает штатно |
| 🟡 | Initializing — устройство инициализируется |
| 🔴 | Error — устройство недоступно или сломано |

---

#### 2.1.2 Главная кнопка системы (Connect Image)

Кнопка отвечает за **запуск и остановку всей системы** (START / STOP).  
Внешний вид меняется в зависимости от текущего состояния:

| Состояние | Текст на кнопке | Изображение |
|---|---|---|
| Базовое (не запущено) | ▶ НАЖМИТЕ ДЛЯ ПОДКЛЮЧЕНИЯ | `media/connect.png` |
| Подключено / работает | ● ПОДКЛЮЧЕНО | `media/green_connect.png` |
| Ошибка | ⚠ ОШИБКА | `media/red_connect.png` |

> При остановке или ручном отключении кнопка возвращается в базовое состояние (`media/connect.png`).

---

#### 2.1.3 Status Label

Текстовая метка под кнопкой. Отражает текущее состояние системы:

| Состояние | Текст | Цвет |
|---|---|---|
| Не запущено (базовое) | ▶ НАЖМИТЕ ДЛЯ ПОДКЛЮЧЕНИЯ | Белый |
| Система готова к работе | ● ПОДКЛЮЧЕНО | Зелёный |
| Обнаружена ошибка | ⚠ ОШИБКА | Красный |

---

#### 2.1.4 Быстрые действия

**Кнопка «Перезапуск»** — располагается в **правом нижнем углу** приложения.  
Перезагружает программу **без её закрытия**.

- Текст кнопки: `Перезапуск`
- Иконка справа от текста (масштабируется под размер текста): `media/reboot.png`

---

#### 2.1.5 Боковое меню — блок состояния (`sidebarBottomBlock`)

Компонент `sidebarBottomBlock` — `QFrame` в `ui/sidebar.py`.  
Отображает оперативную информацию о состоянии программы:

```
STATE:        IDLE / RUNNING / ERROR
MODE:         MANUAL / GESTURE
LAST COMMAND: LEFT
FPS:          24
LATENCY:      68 ms
```

### 2.2 Управление (Control) — ключевая страница

**Назначение:** реальное управление манипулятором в режиме реального времени.

**Layout:**

```
┌─────────────────────────────────────────────────────┐
│                              │                      │
│   Camera Feed   (70%)        │   Control Panel      │
│                              │   (30%)              │
│                              │                      │
└─────────────────────────────────────────────────────┘
```

---

#### Левая часть — CameraView

Видеопоток с камеры.

**Overlay поверх видео** (отображается всегда поверх изображения):

```
[GESTURE:    LEFT  ]
[COMMAND:    MOVE_X -10]
[CONFIDENCE: 0.87  ]
```

---

#### Правая часть — Control Panel

**1. Режим управления**

Переключатель (Toggle):

```
[ Gesture ]  [ Manual ]
```

---

**2. Текущий жест**

```
Gesture:   LEFT
Stability: 92%
```

---

**3. Ручное управление (fallback)**

Кнопки направления:

```
        ↑
   ←         →
        ↓

  [GRAB]   [RELEASE]
```

---

**4. Скорость движения**

```
Speed: [─────●────]   0 — 100
```

---

**5. Чувствительность жестов**

```
Sensitivity: [───●──────]   0 — 100
```

---

**6. ⛔ Emergency Stop (критично)**

```
┌─────────────────────┐
│    !!! STOP !!!     │
└─────────────────────┘
```

При нажатии **мгновенно**:
- останавливает все сервоприводы;
- отключает pipeline.

### 2.3 Настройки (Settings)

**Назначение:** конфигурация системы.

---

#### Блок 1 — Arduino

| Параметр | Тип | Значение по умолчанию |
|---|---|---|

| Gesture | Command |
|---|---|

| RIGHT | MOVE_X +10 |

| OPEN | RELEASE |

> Таблица полностью редактируема пользователем.

- по времени;

## 4. Состояния системы — расширение `AppState`

```python
class SystemState(Enum):
    IDLE        # система ожидает
    CONNECTING  # идёт подключение
    RUNNING     # система работает
    ERROR       # обнаружена ошибка

class ControlMode(Enum):
    GESTURE     # управление жестами
    MANUAL      # ручное управление
```

## 5. UX-сценарии пользователя

### Сценарий 1 — Штатная работа

```
1. Открывает приложение
2. Нажимает «Вход» → попадает на Home
3. Видит статусы: Arduino ❌  Camera ❌
4. Нажимает «Подключить»
5. Видит статусы: Arduino 🟢  Camera 🟢
6. Нажимает «Запустить»
7. Переходит в Control
8. Управляет манипулятором жестами
```

### Сценарий 2 — Ошибка устройства

```
1. Камера не найдена
2. На Home:  🔴 Camera error
3. Кнопка меняется → «Проверить камеру»
4. Пользователь устраняет проблему и повторяет подключение
```

## 6. Ключевые требования к UX

| Требование | Описание |
|---|---|
| Видимость состояния | Статус системы отображается **всегда** — в DeviceBar, Status Label и боковом меню |
| Авторизация | Управление, настройки и журнал доступны **только** авторизованным пользователям |
| Fallback | При недоступности жестового управления — автоматический переход в ручной режим |
| Emergency Stop | Аварийная остановка доступна **в любой момент** из страницы управления |
| Логирование | Все команды и жесты записываются в журнал с временными метками |
